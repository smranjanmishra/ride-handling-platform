# Notification Service

A Spring Boot microservice for managing and sending notifications in a ride-booking system. This service handles various types of notifications including ride acceptance, ride completion, and payment completion.

## Overview

The Notification Service is responsible for:
- Managing notification records in the database
- Sending notifications for ride-related events
- Integrating with the Rider Service to fetch ride details
- Tracking notification history with timestamps

## Features

- ✅ Send ride accepted notifications
- ✅ Send ride completed notifications
- ✅ Send payment completed notifications
- ✅ Store notification history in PostgreSQL
- ✅ RESTful API endpoints
- ✅ Integration with external services (Rider Service)
- ✅ Health checks and metrics (Prometheus)
- ✅ Email notifications support (configurable)
- ✅ JWT authentication support

## Tech Stack

- **Java**: 17

## Installation & Setup

### 1. Clone the Repository

```bash
git clone <repository-url>
cd notification-service
```

### 2. Database Setup

Create a PostgreSQL database:

```sql
CREATE DATABASE notification_db;
```

Or use the provided migration scripts located in `src/main/resources/db/migration/`.

### 3. Configuration

Update `src/main/resources/application.properties` with your configuration:

```properties
# Database Configuration
spring.datasource.url=jdbc:postgresql://localhost:5432/notification_db
spring.datasource.username=postgres
spring.datasource.password=your_password
spring.datasource.driver-class-name=org.postgresql.Driver

# Service URLs
services.driver-service.url=http://localhost:8082
services.rider-service.url=http://localhost:8081

# Email Configuration (Optional)
spring.mail.host=smtp.gmail.com
spring.mail.port=587
spring.mail.username=your-email@gmail.com
spring.mail.password=your-app-password
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
```

### 4. Build the Project

```bash
./mvnw clean install
```

### 5. Run the Application

```bash
./mvnw spring-boot:run
```

The service will start on port **8083** by default.

## API Endpoints

### Base URL
```
http://localhost:8083/api/notifications
```

### 1. Send Ride Accepted Notification

**POST** `/ride-accepted`

Sends a notification when a ride is accepted by a driver.

**Request Body:**
```json
{
  "rideId": 123,
  "driverName": "John Doe"
}
```

**Response:**
```json
{
  "id": 1,
  "userId": 456,
  "rideId": 123,
  "type": "RIDE_ACCEPTED",
  "message": "Your ride #123 has been accepted by driver John Doe",
  "sentAt": "2024-01-15T10:30:00"
}
```

### 2. Send Ride Completed Notification

**POST** `/ride-completed`

Sends a notification when a ride is completed.

**Request Body:**
```json
{
  "userId": 456,
  "rideId": 123
}
```

**Response:**
```json
{
  "id": 2,
  "userId": 456,
  "rideId": 123,
  "type": "RIDE_COMPLETED",
  "message": "Your ride #123 has been completed. Thank you for riding with us!",
  "sentAt": "2024-01-15T11:00:00"
}
```

### 3. Send Payment Completed Notification

**POST** `/payment-completed`

Sends a notification when payment for a ride is completed.

**Request Body:**
```json
{
  "userId": 456,
  "rideId": 123,
  "amount": 250.50
}
```

**Response:**
```json
{
  "id": 3,
  "userId": 456,
  "rideId": 123,
  "type": "PAYMENT_COMPLETED",
  "message": "Payment of ₹250.50 for ride #123 has been completed successfully",
  "sentAt": "2024-01-15T11:05:00"
}
```

## Database Schema

### Notifications Table

```sql
CREATE TABLE notifications (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    user_id BIGINT NOT NULL,
    ride_id BIGINT NOT NULL,
    type VARCHAR(50) NOT NULL,
    message VARCHAR(255) NOT NULL,
    sent_at TIMESTAMP WITHOUT TIME ZONE NOT NULL,
    CONSTRAINT pk_notifications PRIMARY KEY (id)
);
```

**Notification Types:**
- `RIDE_ACCEPTED`: Sent when a driver accepts a ride
- `RIDE_COMPLETED`: Sent when a ride is completed
- `PAYMENT_COMPLETED`: Sent when payment is processed

## Docker Deployment

### Build Docker Image

```bash
docker build \
  --build-arg app=zea-opc-b05-ridebooking-notification-service-soumyami \
  --build-arg version=0.0.1-SNAPSHOT \
  --build-arg lastCommitHash=$(git rev-parse HEAD) \
  --build-arg lastCommitAuthorEmail=$(git log -1 --format='%ae') \
  -t notification-service:latest .
```

### Run Docker Container

```bash
docker run -p 8083:8083 \
  -e SPRING_DATASOURCE_URL=jdbc:postgresql://host.docker.internal:5432/notification_db \
  -e SPRING_DATASOURCE_USERNAME=postgres \
  -e SPRING_DATASOURCE_PASSWORD=your_password \
  notification-service:latest
```

## Kubernetes Deployment
